## Basic Component
### Design:
The `BrowserMockup` component is a React component that simulates a web browser interface within a web application. It incorporates styled components using `styled-components` to create a realistic and interactive browser window. Key styled components include `StyledBrowserWindow`, `BrowserWindow`, `BrowserHeader`, `BrowserButtons`, `BrowserButton`, `BrowserUrlBar`, and `BrowserContent`. These elements collectively form a functional and aesthetically pleasing mock browser. 
### How to behave:
- Input: Users can enter the URL in the address bar to jump
- Display: Users can intuitively see the web page corresponding to the entered URL
### How to use it:
- Installation
Ensure that your project includes all necessary dependencies for the `BrowserMockup` component. If they are not already installed, add them via npm with the following command: 
```
npm install styled-components 
```
- Usage:
To integrate the `styled-components` into your React application, include the necessary imports from the Shadcn UI library:
```
import styled from 'styled-components';

```
- Logic:A state named url is created using the useState hook, with an initial value of "https://www.example.com", which is used to store and update the content of the browser address bar.
The handleUrlChange function handles the input event of the address bar and updates the url state when the user enters the address bar.
```
// BrowserMockup 
const BrowserMockup = () => {
  const [url, setUrl] = useState('https://www.example.com');  
  
  const handleUrlChange = (event) => {
    setUrl(event.target.value); 
  };

  return (
    <StyledBrowserWindow>
      ...
    </StyledBrowserWindow>
  );
};

```


- Customization:
Styled components defined using styled-components allow users to customize the appearance and behavior of the browser emulator by modifying the style properties of these components.

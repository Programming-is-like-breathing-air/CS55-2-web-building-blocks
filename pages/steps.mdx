# Steps Component

**Contributed by Xinyu Chen**

## Basic Steps Component
### Design:

The Steps component provides a visual navigation system for multi-step processes. It includes:

- **Stepper Container**: Visually indicates the progress through steps.
- **Step**: Each step can be styled based on its state (active, completed, or default).
- **Buttons**: "Prev" and "Next" buttons are provided for navigation.

### How it Behaves:

- **Step Interactions**: Clicking on a step sets it as the active step.
- **Navigation**: Users can navigate forward or backward through steps.
- **Dynamic Step Status**: Steps change style based on their state.

### How to Use It:
1. **Import Necessary Components**:
```jsx
import React from "react";
import { Stepper, Step } from "@/components/ui/stepper";
import { Button } from "@/components/ui/button";
```
2. **Setup State Management**:
```tsx
const [activeStep, setActiveStep] = React.useState(0);
const [isLastStep, setIsLastStep] = React.useState(false);
const [isFirstStep, setIsFirstStep] = React.useState(false);
```
3. **Handler Functions**:
```tsx
const handleNext = () => !isLastStep && setActiveStep((cur) => cur + 1);
const handlePrev = () => !isFirstStep && setActiveStep((cur) => cur - 1);
```
4. **Render the Component**:
```tsx
export function StepsBasic() {
    return (
        <div className="w-full py-4 px-8">
            <Stepper
                activeStep={activeStep}
                isLastStep={(value) => setIsLastStep(value)}
                isFirstStep={(value) => setIsFirstStep(value)}
            >
                <Step onClick={() => setActiveStep(0)}>1</Step>
                <Step onClick={() => setActiveStep(1)}>2</Step>
                <Step onClick={() => setActiveStep(2)}>3</Step>
            </Stepper>
            <div className="mt-16 flex justify-between">
                <Button onClick={handlePrev} disabled={isFirstStep}>
                    Prev
                </Button>
                <Button onClick={handleNext} disabled={isLastStep}>
                    Next
                </Button>
            </div>
        </div>
    );
}
```

## With Icon

### Design:

The "Steps with Icon" component enhances the visual representation of each step by incorporating icons. This design provides a more intuitive navigation experience by associating each step with a specific icon.

### How it Behaves:

- **Icon Integration**: Each step is represented by an icon, making it visually distinct and easy to identify.
- **Interactive Icons**: Users can click on the icons directly to navigate between steps.

### How to Use It:

To use the "Steps with Icon" component, follow these steps:

1. **Import Necessary Components and Icons**:

4. **Render the Component with Icons**:

## With Dots
### Design:

The "Steps with Dots" component uses minimalistic dot indicators to represent each step in a process. This design is sleek and visually subtle, making it suitable for clean and modern interfaces where overt icons or text might be distracting.

### How it Behaves:

- **Dot Indicators**: Small dots act as indicators for each step, changing appearance based on their state (active, completed, or default).
- **Clickable Dots**: Each dot is interactive, allowing users to click directly on a dot to navigate to that step.

### How to use it:
The `Step` component automatically renders as a dot unless children (like icons or text) are explicitly provided. This flexibility allows you to use the same component in various configurations throughout your application.

## With Content

### Design:

The "Steps with Content" component enhances the steps by adding titles, subtitles, and icons for each step. This design offers a more informative and engaging experience, ideal for processes where understanding the context of each step is crucial.

### How it Behaves:

- **Rich Content Display**: Each step includes a title, a subtitle with a description, and an associated icon.
- **Interactive Content**: Users can click on any part of the step to navigate to that step, making each step highly interactive.

### How to use it:
You can add text descriptions by adding `title` and `subtitle` directly to the `step` tag.
```tsx
<Step
onClick={() => setActiveStep(2)}
title="Step 3"
subtitle="This is the description of step 3"
isActive={activeStep === 2}
isCompleted={activeStep > 2}
>
<BuildingLibraryIcon className="h-5 w-5" />
</Step>
```

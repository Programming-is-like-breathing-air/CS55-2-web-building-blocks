# Tooltip Component

**Contributed by Yuqing Wu**

A popup that displays information related to an element when the element receives keyboard focus or the mouse hovers over it.

## Basic Tooltip 

### Design:
The `BasicTooltip` component is designed as a versatile React functional component that enables the display of a tooltip adjacent to a button. It uses React's `useState` for state management and is composed of styled components such as `Button`, `Tooltip`, `RightTooltipContent`, `TooltipProvider`, and `TooltipTrigger` from a specific UI library. It is designed to be flexible, allowing users to customize the position of the tooltip relative to the trigger element based on their needs, whether it be to the top, bottom, left, or right. 

### How to behave: 
The interaction with the `BasicTooltip` component is intuitive—hovering over the "Hover" button triggers the visibility of the tooltip. The `handleOpenChange` function toggles the `isOpen` state and logs visibility to the console. The tooltip content, positioned by default to the right as indicated by `RightTooltipContent`, is meant to be customizable, allowing users to position the tooltip in any direction around the button. 

### How to use it: 
- Installation 
Ensure that your project includes all necessary dependencies for the `Tooltip` component. If they are not already installed, add them via npm with the following command: 
``` 
npm shadcn-ui@latest add tooltip
```
- Usuage
To integrate the `Tooltip` component into your React application, include the necessary imports from the Shadcn UI library:
```
import React, { useState } from 'react';
import { Button } from "@/components/ui/button";
import {
  Tooltip,
  RightTooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from "@/components/ui/tooltip";
```
```
<TooltipProvider>
  <Tooltip onOpenChange={handleOpenChange}>
    <TooltipTrigger asChild>
      <Button variant="outline">Hover</Button>
    </TooltipTrigger>
    <RightTooltipContent>
      <p>Add to library</p>
    </RightTooltipContent>
  </Tooltip>
</TooltipProvider>
```
- Customization:
By hovering over the "Hover" button, users activate the tooltip. Developers can modify the tooltip’s position as per their requirement, enabling it to appear to the top, bottom, left, or right of the button. 
```
const RightTooltipContent = React.forwardRef<
  React.ElementRef<typeof TooltipPrimitive.Content>,
  React.ComponentPropsWithoutRef<typeof TooltipPrimitive.Content>
>(({ className, sideOffset = 4, ...props }, ref) => (

  <TooltipPrimitive.Content
    ref={ref}
    //Set the hover on the right
    side="right" 
    sideOffset={5} 
    align="center"
    className={cn(
      "z-50 overflow-hidden rounded-md bg-primary px-3 py-1.5 text-xs text-primary-foreground animate-in fade-in-0 zoom-in-95 data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[side=bottom]:slide-in-from-top-2 data-[side=left]:slide-in-from-right-2 data-[side=right]:slide-in-from-left-2 data-[side=top]:slide-in-from-bottom-2",
      className
    )}
    {...props}
  />
))
RightTooltipContent.displayName = TooltipPrimitive.Content.displayName
```

## Disabled Tooltip

The disabled button with tooltip.

### Design:
The `DisableTooltip` component is a React functional component tailored to present a tooltip for a disabled UI element, symbolized by a button with reduced opacity and a not-allowed cursor style.

### How to behave: 
When users attempt to interact with the button styled to appear disabled ("Hover"), the `handleOpenChange` function is triggered, toggling the `isOpen` state which governs the tooltip's visibility. If the tooltip becomes visible, it logs "Disabled Tooltip is now visible" to the console for diagnostic purposes. This indicates the tooltip is present even for elements that are not active, which can be used to provide additional context or explanations for the disabled state. 

### How to use it: 
- Usuage
To integrate the `Tooltip` component into your React application, include the necessary imports from the Shadcn UI library:
```
    <TooltipProvider>
      <Tooltip onOpenChange={handleOpenChange}>
        <TooltipTrigger asChild>
          <Button variant="outline">Hover</Button>
        </TooltipTrigger>
        <RightTooltipContent>
          <p>Add to library</p>
        </RightTooltipContent>
      </Tooltip>
    </TooltipProvider>
```

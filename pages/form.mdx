## Login Form Component

### Design:
The LoginForm component is designed to facilitate user authentication, providing functionality for both login and signup processes. It features a card-based layout with separate sections for login and signup forms, including input fields for email and password, and additional fields for signup such as confirming the password and accepting terms and conditions. The component utilizes the Card, Input, Label, and Button components for a cohesive and visually appealing user interface. Toast notifications are integrated to provide feedback on login and signup attempts.

### How to behave:
- Login Mode: When in login mode, the user can input their email and password to log in. Upon submitting the login form, a toast notification is displayed indicating a login attempt with the provided email.
- Signup Mode: In signup mode, the user is required to input their email, password, and confirm the password. Additionally, the user must accept the terms and conditions by checking the corresponding checkbox. Upon submitting the signup form, a toast notification is displayed indicating a signup attempt with the provided email.
- Switching Modes: Users can switch between login and signup modes by clicking the appropriate links displayed below the form.

### Checkout Form Component

#### Design:
The CheckoutForm component is designed to facilitate the checkout process for an e-commerce application. It consists of two sections: the "Order Summary" section and the "Payment Details" section. The "Order Summary" section displays the items in the user's cart along with their respective prices and calculates the total cost. The "Payment Details" section allows the user to input their payment information, including card type, card number, cardholder name, expiration date, and CVC. The component utilizes the Card, Input, Label, Select, and Button components for a visually appealing and user-friendly interface.

#### How to behave:
- Order Summary: Displays a list of items in the user's cart along with their prices.
Calculates and displays the total cost of the order.
- Payment Details: Allows the user to select their card type from a dropdown menu. Provides input fields for the card number, cardholder name, expiration date, and CVC. Upon clicking the "Checkout" button, the payment information is submitted for processing.

### Responsive Form Component

#### Design:
The ResponsiveForm component is designed to create a responsive form interface for user input. It includes input fields for the user's first name, last name, country selection, and a textarea for entering a subject. The form is styled with a clean and modern layout, utilizing the Input, Label, Select, and Textarea components for consistent design and functionality.

#### How to behave:
- Input Fields: The "First Name" and "Last Name" fields require users to input their respective names. The "Country" field allows users to select their country from a dropdown menu. The "Subject" field enables users to enter a message or subject for their inquiry.
- Validation: All fields are required, and the form cannot be submitted until all required fields are filled out.

### Form with Multiple Steps
#### Design:
The MultiStepForm component is designed to facilitate a multi-step data entry process, capturing a variety of user information through four distinct steps. The component is structured using the following elements:
- **Card Layout**: Encapsulates the entire form within a visually appealing card interface that includes a header, content section, and footer.
- **Responsive Steps**: The form progresses through four sequential steps, each designed to gather specific types of information: Personal Information, Contact Information, Date of Birth , Account Information


#### How to behave:
- Input Fields: Depending on the step, users are required to input different types of information such as names, contact details, birthday, and account credentials.
- Step Navigation: Users can navigate between steps using "Next" and "Previous" buttons. The form checks the completion of required fields before allowing progression to the next step.
- Validation: Each step includes validation to ensure required fields are not left blank. If validation fails, a toast notification is displayed prompting the user to complete the necessary fields.
- Submission: After filling all fields across the steps and reaching the final step, clicking "Submit" finalizes the data entry. A toast notification confirms the successful registration and welcomes the user.
